# Ink RSVP Spike Validation

Date: 2026-03-05

## Goal

Validate that Ink can drive rapid single-word updates for RSVP-style rendering with:

- `maxFps: 60`
- `incrementalRendering: true`
- update cadence at `120ms` (~500 WPM)

## Script

Run:

```bash
bun run spike:ink
```

Implementation: `scripts/ink-spike.tsx`.

## Result

- Script executes successfully and exits cleanly.
- Alternate screen handling works when a TTY is available.
- In this non-interactive environment, render metrics are not representative of a live terminal.

## Manual TTY Validation Checklist

Run in a real terminal (not CI/non-TTY shell) and verify:

1. No visible flicker during 120ms updates.
2. No layout jumping while word content changes.
3. Alternate screen enters and restores cleanly.
4. No React runtime mismatch errors.

If any item fails, pivot to manual alternate-screen rendering path (already implemented) and consider raw ANSI fallback for the word lane.
